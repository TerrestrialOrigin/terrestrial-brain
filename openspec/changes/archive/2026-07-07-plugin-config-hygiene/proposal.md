## Why

The Obsidian plugin's packaging metadata has drifted: `manifest.json` says version `1.1.0` while `package.json` says `0.3.0`, there is no `versions.json` (Obsidian's mechanism for mapping plugin versions to a minimum app version), the `obsidian` devDependency floats on `"latest"` (non-reproducible builds), and the toolchain (TypeScript, esbuild, `@types/node`) lags the owner's latest-stable rule. Separately, the confirm modal hard-codes ~25 inline `element.style.*` assignments that themes cannot override, and the manifest description is a ~1,400-character wall of usage text that belongs in the README. This is fix-plan Step 27 (findings plugin C1–C3 / "config drift").

## What Changes

- **Version alignment:** make `manifest.json` and `package.json` report the same, single-source-of-truth plugin version.
- **Add `versions.json`** mapping the plugin version to its `minAppVersion`, per Obsidian's plugin-distribution convention.
- **Pin the `obsidian` devDependency** to an explicit stable version instead of `"latest"` for reproducible builds.
- **Bump the toolchain** (TypeScript, esbuild, `@types/node`) to current stable versions per the owner's latest-stable rule, keeping the build and tests green.
- **Trim the manifest `description`** to one or two sentences; move the command/usage/exclude-tag detail into the README.
- **Move the ~25 inline `element.style.*` assignments** out of `src/confirmModal.ts` into a root-level `styles.css`, keyed on the existing `tb-ai-output-*` CSS classes so themes can override them.
- **Remove the redundant `<select>` option `value` re-assignment** (the `createEl` `value` option already sets it).

## Non-goals

- No change to plugin runtime behavior, sync logic, API surface, or settings. This is packaging/hygiene only; the confirm modal must render and behave identically.
- No visual redesign of the confirm modal — styles move verbatim from inline assignments to `styles.css`.
- No changes to the Deno/MCP backend, migrations, or the server test suite.
- Not publishing the plugin to the Obsidian community registry (that is out of scope for this step).
